# Python Trading Bot (Binance Futures Testnet)

CLI-based Python trading bot for Binance USDT-M Futures Testnet.

## Phase Status

- Phase 1 completed: setup and scaffold
- Phase 2 completed: core implementation
- Phase 3 completed: execution and validation tests
- Phase 4 in progress: documentation and packaging

## Features

- MARKET and LIMIT order placement
- Input validation before API calls
- Structured logging for requests, responses, and errors
- Modular architecture by responsibility

## Project Structure

- `bot/client.py` - Binance Futures testnet client initialization
- `bot/orders.py` - MARKET/LIMIT order execution
- `bot/validators.py` - Input normalization and validation
- `bot/logging_config.py` - File + console logger setup
- `bot/cli.py` - argparse-based command line entrypoint
- `.env` - API credentials (not committed)
- `.gitignore` - excludes secrets and Python artifacts
- `requirements.txt` - Python dependencies
- `logs/trading.log` - runtime log output
- `TEST_REPORT.md` - phase 3 test evidence summary

## Requirements

- Python 3.10+
- Binance Futures Testnet account with API key/secret

## Setup

1. Create and activate virtual environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. Create `.env` in project root:

```env
BINANCE_API_KEY=your_testnet_key
BINANCE_API_SECRET=your_testnet_secret
```

## Usage

Run from project root.

### MARKET order

```bash
python bot/cli.py --symbol BTCUSDT --side BUY --type MARKET --quantity 0.001
```

### LIMIT order

```bash
python bot/cli.py --symbol BTCUSDT --side SELL --type LIMIT --quantity 0.001 --price 90000
```

## CLI Arguments

- `--symbol` required, uppercase pair like `BTCUSDT`
- `--side` required, `BUY` or `SELL`
- `--type` required, `MARKET` or `LIMIT`
- `--quantity` required, must be `> 0`
- `--price` required only when `--type LIMIT`, must be `> 0`

## Logging

- Log file path: `logs/trading.log`
- Logged events:
	- validated inputs
	- API request payload
	- API response
	- validation and execution errors

## Validation/Error Behavior

- Invalid input returns exit code `2`
- Execution/API failures return exit code `1`
- Errors are printed to CLI and written to `logs/trading.log`

- Bot targets Binance USDT-M Futures Testnet only.

- User has testnet balance and symbol enabled.
